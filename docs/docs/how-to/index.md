# How-To Guides

Task-oriented guides for accomplishing specific goals with CALM. Each guide is standalone and assumes you have basic CALM knowledge.

:::tip New to CALM?
If you're just getting started, begin with the [Tutorials](../tutorials/) first to learn the fundamentals.
:::

## Modeling

Guides for modeling different aspects of your architecture:

| Guide | Description |
|-------|-------------|
| [Define Controls](modeling/controls) | Add non-functional requirements to your architecture |
| [Model Business Flows](modeling/flows) | Document how processes flow through your system |
| [Document Decisions with ADRs](modeling/adrs) | Record architecture decisions and their rationale |

Guides for generating and customizing documentation:

## Quick Reference

### When to Use Each Guide

| If you want to... | See |
|-------------------|-----|
| Add security/performance requirements | [Controls](modeling/controls) |
| Show how a request flows through services | [Business Flows](modeling/flows) |
| Record why you chose a technology | [ADRs](modeling/adrs) |
| Auto-generate Markdown docs | [Docify](documentation/docify) |
| Create reusable doc templates | [Custom Widgets](documentation/custom-widgets) |
| Create ops runbooks | [Operations Docs](operations/ops-docs) |
| Enforce architecture standards | [Define Standards](governance/standards) |
| Validate in CI/CD | [Multi-Pattern Validation](governance/multi-pattern-validation) |
