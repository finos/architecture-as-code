## Decisions Made

- AIGF data inlined as TypeScript constants (not JSON imports) — `tsconfig.base.json` lacks `resolveJsonModule`, so JSON imports would fail at compile time
- `ai:guardrail`, `ai:human-in-the-loop`, `ai:eval-monitor` correctly have empty `applicableRisks` arrays — these ARE the mitigations in AIGF v2.0 design philosophy
- `isAINode` accepts union type `(string | {node-type: string})` — ergonomic for callers using either raw node objects or type strings
- `metadata?: Record<string, unknown>` (not `Record<string, string>`) — nested structured data needed for AIGF decorator payloads
