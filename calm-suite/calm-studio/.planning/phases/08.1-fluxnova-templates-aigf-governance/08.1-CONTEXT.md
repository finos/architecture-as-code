<decisions>
## Implementation Decisions

### Template Picker Experience
- Full-screen modal overlay for maximum visual impact at OSFF demo
- Category tabs at top: 'FluxNova', 'AI Governance', 'General'
- Compact cards with category icon + template name + one-line description
- Accessible via dedicated 'Templates' toolbar button AND 'Start from template' link on empty canvas
- Canvas dirty state prompts confirmation before template load
- `_template` metadata stripped on CALM JSON export

### Template Content
- All 6 templates from the PRD to be created (2 already exist):
  - fluxnova-platform.json (exists)
  - fluxnova-kyc-onboarding.json (exists)
  - fluxnova-flash-risk.json (to create)
  - fluxnova-settlement.json (to create)
  - fluxnova-ai-agent.json (to create)
  - fluxnova-microservices.json (to create)

### Governance Panel Placement
- Right sidebar tab alongside the properties panel (new 'Governance' tab)
- Manual tab switch only — no auto-switch when AI nodes are selected (respects opt-in UX preference)
- Subtle badge/dot on Governance tab indicates when governance info is available for the selected node
- Two-section layout: Risks section (top) with severity badges (OP=amber, SEC=red, RC=blue), then Mitigations section (bottom) with 'Apply' buttons
- Both architecture-level summary in panel header (governance score + unmitigated count) AND per-node detail for selected node

### External Framework References
- Show top 3 most relevant framework refs inline per risk/mitigation (EU AI Act, ISO 42001, OWASP LLM Top 10)
- 'Show all' expander reveals full 7-framework mapping (NIST, FFIEC, MITRE ATLAS, etc.)

### Controls UX in Properties Panel
- Collapsible 'Controls' section in both NodeProperties and EdgeProperties
- Each control shown as collapsible row: control key as heading, description below, requirement-url as clickable link
- Applied controls are editable; new controls added primarily via governance panel 'Apply'
- Controls on both nodes AND relationships (not nodes only)

### Data Classification on Canvas
- Small colored tag/pill below node name on canvas: 'PII' in red, 'Confidential' in amber, 'Public' in green
- Visible at a glance without selecting the node

### Governance Score
- Architecture-level score shown in governance panel header AND as colored percentage badge in toolbar
- Badge coloring: green (>80%), amber (50-80%), red (<50%)
- Toolbar badge hidden when architecture has no AI nodes (keeps toolbar clean for non-AI architectures)
- Score updates live when controls are applied/removed (NOT tied to explicit Validate trigger — this is a counter, not a validation run)

### CALM 1.2 Spec Support
- **Controls:** Full support on nodes and relationships per CALM 1.2 control.json schema. Keys use `aigf-` prefix for AIGF-specific controls.
- **Decorators:** Generate AIGF governance decorator on export — architecture-wide overlay summarizing assessed risks, governance score, regulatory mappings. CalmGuard reads this for reporting.
- **Evidence:** Types added to calm-core for roundtrip completeness. Evidence collection/linking is CalmGuard's responsibility at build/runtime, not CalmStudio design-time.

### Claude's Discretion
- Whether properties panel includes manual 'Add Control' button (for custom non-AIGF controls) vs governance 'Apply' only — decide based on implementation complexity and demo needs
- Exact governance panel component layout and styling details
- AIGF validation rule severity tuning (which rules are errors vs warnings vs info)
- Template card hover/selection interaction patterns
- Decorator generation logic details (which fields to include, when to generate)

</decisions>

<specifics>
## Specific Ideas

- AIGF is building the reference CALM integration — nobody has done this yet. GitHub issue #139 in finos/ai-readiness explicitly plans it. CalmStudio is the first implementation.
- FluxNova is a FINOS Camunda 7 fork used by Fidelity, NatWest, Deutsche Bank, Capital One — the templates need to tell that enterprise financial services story.
- The OSFF Toronto demo has both a FluxNova workshop AND an AIGF training workshop on the same day — CalmStudio bridges both with the ai-agent template (FluxNova orchestrating AI agents with AIGF governance).
- The governance chain: Requirement Schema → Control Detail → Controls Object → CalmGuard validates → Evidence → Decorators overlay. CalmStudio operates at Controls + Decorators (design-time).
- AIGF_CATALOGUE.json already has all 23 risks + 23 mitigations structured — import directly, don't manually re-type.
- Templates already use CALM 1.2 controls pattern: `controls: { "domain-key": { description, requirements: [{ requirement-url, config? }] } }`

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- **Extension pack system** (`packages/extensions/`): PackDefinition type, registerPack(), resolvePackNode(), initAllPacks() — FluxNova pack follows exact same pattern as ai.ts/aws.ts
- **Pack icon pattern** (`packages/extensions/src/icons/`): Hand-crafted 16x16 SVG strings, stroke-based, fill="none", currentColor
- **ValidationPanel** (`apps/studio/src/lib/validation/`): Bottom drawer with severity sorting, two-way navigation — AIGF rules wire into existing store
- **NodeProperties/EdgeProperties** (`apps/studio/src/lib/properties/`): Debounced mutation pattern with onBeforeFirstEdit for undo coordination — controls section follows same pattern
- **calmModel.svelte.ts** (`apps/studio/src/lib/stores/`): updateNodeProperty() and updateEdgeProperty() for applying controls via mutations
- **Toolbar** (`apps/studio/src/lib/toolbar/`): $props() callback pattern — add Templates button and governance badge
- **Import/export** (`apps/studio/src/lib/io/`): File I/O functions — extend to strip _template metadata on export and generate decorators

### Established Patterns
- Module-level $state() with accessor/mutation functions for stores (no class instances)
- Debounced property updates (300ms) with first-edit signaling for undo history
- Pack registration at startup via initAllPacks() in both NodePalette and +page.svelte
- Validation is on-demand (explicit Validate button), not automatic
- Svelte 5 $props() rune for all component interfaces

### Integration Points
- `packages/calm-core/src/types.ts` — Add CalmControls, CalmControl, CalmControlRequirement, CalmDecorator, CalmEvidence types
- `packages/calm-core/src/validation.ts` — Add control key validation (^[a-zA-Z0-9-]+$)
- `packages/extensions/src/index.ts` — Register fluxnovaPack in initAllPacks()
- `apps/studio/src/lib/toolbar/Toolbar.svelte` — Add Templates button + governance badge
- `apps/studio/src/lib/properties/NodeProperties.svelte` — Add controls collapsible section (replace Phase 6 placeholder)
- `apps/studio/src/lib/properties/EdgeProperties.svelte` — Add controls collapsible section
- `apps/studio/src/lib/stores/validation.svelte.ts` — Aggregate AIGF rules with structural rules in runValidation()
- `apps/studio/src/routes/+page.svelte` — Wire governance panel, template picker, and right sidebar tabs

</code_context>
