<task type="auto">
  <name>Task 1: Add readonly mode and ondblclicknode to CalmCanvas</name>
  <files>apps/studio/src/lib/canvas/CalmCanvas.svelte</files>
  <action>
    Modify CalmCanvas.svelte to support read-only C4 navigation mode:

    1. **Add new props** to the `$props()` destructure:
       - `readonly = false` (boolean, default false)
       - `ondblclicknode?: (node: Node) => void` (callback for C4 drill-down)

    2. **Pass readonly to SvelteFlow** — on the `<SvelteFlow>` element, add:
       - `nodesDraggable={!readonly}`
       - `nodesConnectable={!readonly}`
       - `deleteKey={readonly ? [] : ['Delete', 'Backspace']}`
       - Keep `elementsSelectable` as-is (selection still works in readonly — per CONTEXT: "Properties panel shows selected node info but read-only")

    3. **Add onnodedblclick handler** to `<SvelteFlow>`:
       ```
       onnodedblclick={(e) => {
         if (readonly && ondblclicknode) {
           ondblclicknode(e.node);
         }
       }}
       ```
       Note: SvelteFlow in @xyflow/svelte fires `onnodedblclick` — verify exact prop name from the existing SvelteFlow type imports. If it's `onNodeDoubleClick` or similar, use that.

    4. **Gate mutation handlers** — add `if (readonly) return;` at the top of:
       - `handleDrop` (no new nodes in readonly)
       - `handleConnect` (no new edges in readonly)
       - `handleNodeDragStop` (no containment changes in readonly)
       - `handleCopy` / `handlePaste` (no clipboard ops in readonly)
       - `handleUndo` / `handleRedo` (no undo/redo in readonly — the model isn't being changed)
       Do NOT gate `handleSelectionChange` — selection still works for properties inspection.

    5. **Gate oncanvaschange** — any place that calls `oncanvaschange?.()` should check `if (readonly) return` first, OR wrap in a helper: `function notifyChange() { if (!readonly) oncanvaschange?.(); }`. This prevents isDirty from becoming true during C4 browsing (Pitfall 2).

    6. **Conditionally render interaction elements** — the edge context menu (right-click to change edge type) should not appear in readonly mode. Gate `handleEdgeContextMenu` with `if (readonly) return;`.
  </action>
