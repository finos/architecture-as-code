  <action>
    Create `apps/studio/src/lib/c4/` directory.

    Create `c4Filter.ts` as a pure TypeScript file (NO .svelte.ts imports — same pattern as projection.ts and containment.ts):

    1. Export `type C4Level = 'context' | 'container' | 'component'`

    2. Define `CONTEXT_TYPES = new Set(['actor', 'system', 'ecosystem'])` and `CONTAINER_TYPES = new Set(['service', 'database', 'webclient', 'network', 'ldap', 'data-asset'])` — per CONTEXT.md locked decisions.

    3. `classifyNodeC4Level(calmType: string): C4Level` — Set.has() lookup, default to 'component' for everything else (aws:*, gcp:*, azure:*, k8s:*, ai:*, generic, any custom type).

    4. `isExternalNode(node: { data?: Record<string, unknown> }): boolean` — check `data.customMetadata['c4-scope'] === 'external'` OR `data.calmType === 'ecosystem'`. Use defensive access (customMetadata may be undefined).

    5. `filterNodesForLevel(nodes, level, drillParentId)`:
       - If drillParentId is non-null: return nodes where `node.parentId === drillParentId`
       - If drillParentId is null:
         - 'context': nodes where classifyNodeC4Level(node.data.calmType) === 'context' AND `!node.parentId` (Pitfall 3 — only top-level nodes)
         - 'container': nodes where classifyNodeC4Level === 'container' AND `!node.parentId`
         - 'component': nodes where classifyNodeC4Level === 'component' AND `!node.parentId`

    6. `filterEdgesForVisibleNodes(edges, visibleIds: Set<string>): Edge[]` — return edges where both `source` and `target` are in `visibleIds`.

    7. `getChildrenOf(parentId: string, nodes): Node[]` — filter by `node.parentId === parentId`.

    8. `hasDrillableChildren(nodeId: string, nodes): boolean` — `nodes.some(n => n.parentId === nodeId)`.

    9. `applyC4Styles(nodes, level: C4Level): Node[]` — map over nodes, inject `data.c4Level = level` and `data.c4External = isExternalNode(node)` into each node's data. Use shallow spread `{ ...node, data: { ...node.data, c4Level, c4External } }` (not deep clone).

    Write test file at `apps/studio/src/tests/c4Filter.test.ts` FIRST (RED), then implement (GREEN). Use vitest `describe`/`it` blocks. Create minimal mock nodes/edges for test fixtures. Follow `apps/studio/src/tests/` existing test patterns.

    SPDX headers on both files: `// SPDX-FileCopyrightText: 2026 CalmStudio Contributors` + `// SPDX-License-Identifier: Apache-2.0`
