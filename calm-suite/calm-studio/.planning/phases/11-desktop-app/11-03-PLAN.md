  <action>
    Two parts: (A) configure the MCP server to build as a standalone sidecar binary, and (B) create the GitHub Actions workflow for cross-platform desktop builds.

    **Part A: MCP Server Sidecar Build**

    1. Add `@yao-pkg/pkg` as a devDependency to `packages/mcp-server/`:
       ```bash
       cd packages/mcp-server && pnpm add -D @yao-pkg/pkg esbuild
       ```

    2. Add sidecar build scripts to `packages/mcp-server/package.json`:
       ```json
       "build:bundle": "esbuild src/index.ts --bundle --platform=node --format=cjs --outfile=dist/bundle.cjs --external:elkjs-svg",
       "build:sidecar": "pnpm build:bundle && pkg dist/bundle.cjs --targets node20-macos-arm64,node20-macos-x64,node20-linux-x64,node20-win-x64 --output dist/calmstudio-mcp"
       ```
       IMPORTANT: The MCP server uses ESM (`"type": "module"`). `@yao-pkg/pkg` requires CJS input. Use esbuild to bundle to CJS first, then pkg to create standalone binaries. This avoids the ESM pitfall documented in RESEARCH Pitfall 6.
       NOTE: `elkjs-svg` is marked external because it may have native dependencies; test and include if needed.

    3. Add a `build:sidecar-local` script for dev convenience that builds only the current platform:
       ```json
       "build:sidecar-local": "pnpm build:bundle && pkg dist/bundle.cjs --targets node20-macos-arm64 --output dist/calmstudio-mcp"
       ```

    4. Document the sidecar binary naming convention. Tauri requires exact target triple suffixes (RESEARCH Pitfall 2):
       - macOS ARM: `calmstudio-mcp-aarch64-apple-darwin`
       - macOS Intel: `calmstudio-mcp-x86_64-apple-darwin`
       - Linux: `calmstudio-mcp-x86_64-unknown-linux-gnu`
       - Windows: `calmstudio-mcp-x86_64-pc-windows-msvc.exe`

       The CI workflow handles renaming. For local dev, create a script in `apps/studio/scripts/copy-sidecar.sh` that copies the pkg output to `src-tauri/binaries/` with the correct triple name. Alternatively, add this as a `build:sidecar-copy` script.

    5. Create `apps/studio/src-tauri/binaries/.gitkeep` so the directory exists.

    **Part B: GitHub Actions Desktop Build Workflow**

    Create `.github/workflows/desktop-build.yml`:

    ```yaml
    name: Desktop Build

    on:
      push:
        tags:
          - 'v*'
      workflow_dispatch:
        inputs:
          draft:
            description: 'Create as draft release'
            type: boolean
            default: true

    permissions:
      contents: write

    jobs:
      build-sidecar:
        name: Build MCP Sidecar (${{ matrix.target }})
        runs-on: ${{ matrix.os }}
        strategy:
          matrix:
            include:
              - os: macos-latest
                target: node20-macos-arm64
                triple: aarch64-apple-darwin
              - os: macos-13
                target: node20-macos-x64
                triple: x86_64-apple-darwin
              - os: ubuntu-22.04
                target: node20-linux-x64
                triple: x86_64-unknown-linux-gnu
              - os: windows-latest
                target: node20-win-x64
                triple: x86_64-pc-windows-msvc
        steps:
          - uses: actions/checkout@v4
          - uses: pnpm/action-setup@v4
            with: { version: 9 }
          - uses: actions/setup-node@v4
            with: { node-version: '20', cache: 'pnpm' }
          - run: pnpm install --frozen-lockfile
          - run: pnpm --filter @calmstudio/calm-core build
          - name: Build sidecar
            run: |
              cd packages/mcp-server
              pnpm build:bundle
              pnpm dlx @yao-pkg/pkg dist/bundle.cjs --target ${{ matrix.target }} --output dist/calmstudio-mcp
          - name: Rename with target triple
            shell: bash
            run: |
              cd packages/mcp-server/dist
              if [ "${{ runner.os }}" = "Windows" ]; then
                mv calmstudio-mcp.exe calmstudio-mcp-${{ matrix.triple }}.exe
              else
                mv calmstudio-mcp calmstudio-mcp-${{ matrix.triple }}
              fi
          - uses: actions/upload-artifact@v4
            with:
              name: sidecar-${{ matrix.triple }}
              path: packages/mcp-server/dist/calmstudio-mcp-*

      build-desktop:
        name: Build Desktop (${{ matrix.triple }})
        needs: build-sidecar
        runs-on: ${{ matrix.os }}
        strategy:
          matrix:
            include:
              - os: macos-latest
                args: --target aarch64-apple-darwin
                triple: aarch64-apple-darwin
              - os: macos-13
                args: --target x86_64-apple-darwin
                triple: x86_64-apple-darwin
              - os: ubuntu-22.04
                args: ''
                triple: x86_64-unknown-linux-gnu
              - os: windows-latest
                args: ''
                triple: x86_64-pc-windows-msvc
        steps:
          - uses: actions/checkout@v4
          - uses: pnpm/action-setup@v4
            with: { version: 9 }
          - uses: actions/setup-node@v4
            with: { node-version: '20', cache: 'pnpm' }
          - run: pnpm install --frozen-lockfile

          - name: Install Linux dependencies
            if: runner.os == 'Linux'
            run: |
              sudo apt-get update
              sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf

          - name: Install Rust
            uses: dtolnay/rust-toolchain@stable
            with:
              targets: ${{ matrix.triple }}

          - name: Rust cache
            uses: swatinem/rust-cache@v2
            with:
              workspaces: apps/studio/src-tauri

          - name: Download sidecar binary
            uses: actions/download-artifact@v4
            with:
              name: sidecar-${{ matrix.triple }}
              path: apps/studio/src-tauri/binaries/

          - name: Make sidecar executable
            if: runner.os != 'Windows'
            run: chmod +x apps/studio/src-tauri/binaries/calmstudio-mcp-*

          - name: Build desktop app
            uses: tauri-apps/tauri-action@v0
            env:
              GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
              APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
              APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
              APPLE_SIGNING_IDENTITY: ${{ secrets.APPLE_SIGNING_IDENTITY }}
              APPLE_ID: ${{ secrets.APPLE_ID }}
              APPLE_PASSWORD: ${{ secrets.APPLE_PASSWORD }}
              APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
              TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
              TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
            with:
              projectPath: apps/studio
              tauriScript: pnpm tauri
              args: ${{ matrix.args }}
              tagName: ${{ github.ref_name }}
              releaseName: 'CalmStudio ${{ github.ref_name }}'
              releaseBody: 'See the [changelog](https://github.com/${{ github.repository }}/blob/main/CHANGELOG.md) for details.'
              releaseDraft: ${{ github.event.inputs.draft || 'false' }}
              prerelease: false
    ```

    Key design decisions per CONTEXT.md locked decisions:
    - **Triggered on git tag push** (`v*`) per "CI builds on git tag push"
    - **DMG (macOS), NSIS (Windows), AppImage (Linux)** — Tauri default bundle targets
    - **macOS notarization** — Apple signing secrets are optional; build succeeds unsigned
    - **Auto-update artifacts** — `createUpdaterArtifacts: true` produces `latest.json`
    - **Sidecar built first** in separate job, then downloaded as artifact

    6. **Update tauri.conf.json updater endpoint** to match the actual GitHub org/repo:
       Replace the placeholder in `plugins.updater.endpoints` with:
       ```
       "https://github.com/finos/calmstudio/releases/latest/download/latest.json"
       ```
       (Or whatever the actual GitHub URL is — the executor should check the git remote.)

    AVOID: Do NOT modify the existing `release.yml` (npm package releases). Desktop builds are a separate workflow.
    AVOID: Do NOT require Apple signing secrets for CI to pass. They are optional — builds work unsigned.
