  <action>
    Create all 6 desktop feature modules. Each module is self-contained and exports a focused API.

    1. **`menu.ts`** — Native menu bar following Pattern 3 from RESEARCH:
       - Export `MenuHandlers` interface: `{ open, save, saveAs, newFile, exportCalm, exportSvg, exportPng, undo, redo, zoomIn, zoomOut, zoomFit, togglePalette, toggleCode, toggleProperties, about, docs }` — all `() => void | Promise<void>`.
       - Export `buildAppMenu(handlers: MenuHandlers): Promise<void>`.
       - Build menu structure using `Menu, MenuItem, Submenu, PredefinedMenuItem` from `@tauri-apps/api/menu`.
       - **File menu:** Open (CmdOrCtrl+O), Save (CmdOrCtrl+S), Save As (CmdOrCtrl+Shift+S), Separator, Recent Files submenu (populated dynamically), Separator, New (CmdOrCtrl+N).
       - **Edit menu:** Undo (PredefinedMenuItem), Redo (PredefinedMenuItem), Separator, Cut (PredefinedMenuItem), Copy (PredefinedMenuItem), Paste (PredefinedMenuItem).
       - **View menu:** Zoom In (CmdOrCtrl+=), Zoom Out (CmdOrCtrl+-), Zoom to Fit (CmdOrCtrl+0), Separator, Toggle Palette, Toggle Code Panel, Toggle Properties.
       - **Help menu:** About CalmStudio, Documentation (opens https://calmstudio.dev/docs in browser).
       - Export `updateRecentFilesMenu(recentPaths: string[]): Promise<void>` to rebuild the recent files submenu dynamically. Each recent file item triggers the file open handler with the path.
       - macOS constraint: all items in submenus (no top-level loose items).
       - Accelerators use CmdOrCtrl for cross-platform.

    2. **`recentFiles.ts`** — Recent files list following Pattern 7 from RESEARCH:
       - Export `addRecentFile(path: string): Promise<string[]>` — adds path to front, deduplicates, caps at 10.
       - Export `getRecentFiles(): Promise<string[]>` — returns list.
       - Export `clearRecentFiles(): Promise<void>` — clears list.
       - Uses `load('settings.json', { autoSave: true })` from `@tauri-apps/plugin-store`.
       - Key in store: `'recentFiles'`.

    3. **`sidecarMcp.ts`** — MCP sidecar lifecycle following Pattern 5 from RESEARCH:
       - Export `startMcpSidecar(): Promise<void>` — spawns sidecar via `Command.sidecar('binaries/calmstudio-mcp')`.
       - Export `stopMcpSidecar(): Promise<void>` — kills child process.
       - Store child reference in module-level variable.
       - IMPORTANT: Wrap `spawn()` in try/catch. If sidecar binary is missing (dev mode, no pkg binary built yet), log warning to console but do NOT crash the app.

    4. **`dragDrop.ts`** — Drag-and-drop handler following RESEARCH code example:
       - Export `registerFileDrop(onFile: (path: string) => void): () => void`.
       - Uses `getCurrentWebview().onDragDropEvent()` from `@tauri-apps/api/webview`.
       - On `drop` event: find first path ending in `.json` or `.calm.json`, call `onFile(path)`.
       - Return unlisten function for cleanup.
       - Add 50ms debounce per RESEARCH open question #3 (drag-drop event duplication bug in Tauri 2.8.4).

    5. **`updater.ts`** — Auto-update check following Pattern 6 from RESEARCH:
       - Export `checkForUpdates(): Promise<void>`.
       - Uses `check()` from `@tauri-apps/plugin-updater` and `relaunch()` from `@tauri-apps/plugin-process`.
       - If update available: use `window.confirm()` to ask user (simple, no custom UI needed).
       - If confirmed: `await update.downloadAndInstall()` then `await relaunch()`.
       - Wrap entire function in try/catch — never crash on update failure. Log errors to console.

    6. **`fileOpen.ts`** — Handle file-open events from OS (double-click, single-instance, macOS cold-start):
       - Export `registerFileOpenHandler(onFile: (path: string) => void): () => void`.
       - **Single-instance events (Windows/Linux warm-start):** Listen for `'open-file'` event from Tauri (emitted by single-instance plugin in lib.rs when a second instance tries to launch with a file path). Uses `listen()` from `@tauri-apps/api/event`.
       - **macOS cold-start:** On macOS, when the app is launched fresh by double-clicking a .calm.json file, the single-instance plugin does NOT fire (there is no existing instance). Instead, use `getCurrent()` from `@tauri-apps/plugin-deep-link` to check if the app was launched with a file URL. Call `getCurrent()` once during registration — if it returns URLs, extract the first file path and call `onFile(path)`. This handles the macOS Finder "open with" and file association cold-start case (see RESEARCH Pitfall 4).
       - Return unlisten function for cleanup (only the event listener needs cleanup; getCurrent is a one-shot call).

    AVOID: Do NOT import Svelte stores or $state in these modules. They are pure TypeScript — they receive callbacks and return promises. The Svelte integration happens in +page.svelte.
    AVOID: Do NOT block app startup. Sidecar and updater run async fire-and-forget.

    2. **Create desktop file-open helper** inside the `<script>`:
       ```typescript
       async function handleOpenFromPath(path: string) {
         try {
           const content = await readTextFile(path);
           const name = path.split(/[\\/]/).pop() ?? path;
           await importCalmFile(content, name);
           markClean(name, path);
           await addRecentFile(path);
           const recent = await getRecentFiles();
           await updateRecentFilesMenu(recent);
         } catch (e) {
           console.error('Failed to open file from path:', e);
         }
       }
       ```

    3. **Add desktop onMount block.** Find the existing `onMount` (if any) or add a new one. Inside, guard with `if (isTauri())`:
       ```typescript
       onMount(() => {
         if (!isTauri()) return;

         let cleanups: (() => void)[] = [];

         // 1. Build native menu bar
         const wrappedHandleOpen = async () => {
           await handleOpen();
           // After successful open, add to recent files
           const path = getFileHandle();
           if (typeof path === 'string') {
             await addRecentFile(path);
             const recent = await getRecentFiles();
             await updateRecentFilesMenu(recent);
           }
         };
         buildAppMenu({
           open: wrappedHandleOpen,
           save: handleSave,
           saveAs: handleSaveAs,
           newFile: handleNew,
           exportCalm: handleExportCalm,
           exportSvg: handleExportSvg,
           exportPng: handleExportPng,
           undo: () => { /* call undo from history store */ },
           redo: () => { /* call redo from history store */ },
           zoomIn: () => { /* dispatch to canvas */ },
           zoomOut: () => { /* dispatch to canvas */ },
           zoomFit: () => { /* dispatch to canvas */ },
           togglePalette: () => { /* toggle palette visibility */ },
           toggleCode: () => { /* toggle code panel */ },
           toggleProperties: () => { /* toggle properties panel */ },
           about: () => { alert('CalmStudio v0.1.0\nVisual CALM Architecture Editor\nhttps://calmstudio.dev'); },
           docs: () => { window.open('https://calmstudio.dev/docs', '_blank'); },
         });

         // 2. Register drag-and-drop
         const unlistenDrop = registerFileDrop(handleOpenFromPath);
         cleanups.push(unlistenDrop);

         // 3. Register file-open events (from single-instance / deep-link)
         // This handles both:
         //   - Windows/Linux: single-instance plugin emitting 'open-file' event
         //   - macOS cold-start: deep-link getCurrent() one-shot check
         const unlistenFileOpen = registerFileOpenHandler(handleOpenFromPath);
         cleanups.push(unlistenFileOpen);

         // 4. Start MCP sidecar (fire-and-forget)
         startMcpSidecar().catch(e => console.warn('MCP sidecar failed to start:', e));

         // 5. Check for updates (fire-and-forget, never blocks)
         checkForUpdates().catch(e => console.warn('Update check failed:', e));

         // 6. Populate recent files in menu
         getRecentFiles().then(recent => {
           if (recent.length > 0) {
             updateRecentFilesMenu(recent);
           }
         });

         return () => {
           cleanups.forEach(fn => fn());
           stopMcpSidecar().catch(() => {});
         };
       });
       ```

    4. **Wire undo/redo from menu to history store.** The existing `undo` and `redo` functions from `$lib/stores/history.svelte` should be imported and passed to the menu handlers. Check the existing imports and wire `undo` and `redo` directly.

    5. **Wire zoom handlers.** If the canvas component exposes `zoomIn/zoomOut/fitView` via `bind:this`, wire those. Otherwise, use the Svelte Flow instance methods.     AVOID: Do NOT restructure the existing +page.svelte significantly. Add the desktop block as a focused addition.
    AVOID: Do NOT make the menu handlers depend on Svelte reactivity. They receive plain functions, not reactive stores.
