1. **Create `.github/workflows/` directory structure.**

2. **.github/workflows/ci.yml** — Runs on `push` to main and `pull_request` to main. Four jobs:

   **Job: build-lint-test** (ubuntu-latest)
   - `actions/checkout@v4`
   - `pnpm/action-setup@v4` with `version: 9`
   - `actions/setup-node@v4` with `node-version: '20'`, `cache: 'pnpm'`
   - `pnpm install --frozen-lockfile`
   - `pnpm -r run build`
   - `pnpm -r run lint`
   - `pnpm -r run test`

   **Job: reuse-compliance** (ubuntu-latest)
   - `actions/checkout@v4`
   - `fsfe/reuse-action@v6`

   **Job: cve-scan** (ubuntu-latest)
   - `actions/checkout@v4`
   - `pnpm/action-setup@v4` with `version: 9`
   - `actions/setup-node@v4` with `node-version: '20'`, `cache: 'pnpm'`
   - `pnpm install --frozen-lockfile`
   - OWASP Dependency-Check using `dependency-check/Dependency-Check_Action@main`:
     - `project: 'calmstudio'`
     - `path: '.'`
     - `format: 'HTML'`
     - `args: '--failOnCVSS 7 --enableRetired --nvdApiKey ${{ secrets.NVD_API_KEY }}'`
   - Upload OWASP report artifact with `actions/upload-artifact@v4`

   **Job: commitlint** (ubuntu-latest) — validates PR commits:
   - `actions/checkout@v4` with `fetch-depth: 0`
   - `pnpm/action-setup@v4` with `version: 9`
   - `actions/setup-node@v4` with `node-version: '20'`, `cache: 'pnpm'`
   - `pnpm install --frozen-lockfile`
   - `pnpm exec commitlint --from ${{ github.event.pull_request.base.sha }} --to ${{ github.event.pull_request.head.sha }}` (only on pull_request event)

   Set `permissions: contents: read` at workflow level.

3. **.github/workflows/release.yml** — Runs on `push` to `main` only:
   - `actions/checkout@v4` with `fetch-depth: 0` and `persist-credentials: false` (CRITICAL: fetch-depth 0 needed per Pitfall 3)
   - `pnpm/action-setup@v4` with `version: 9`
   - `actions/setup-node@v4` with `node-version: '20'`, `cache: 'pnpm'`, `registry-url: 'https://registry.npmjs.org'`
   - `pnpm install --frozen-lockfile`
   - `pnpm -r run build`
   - `pnpm dlx multi-semantic-release`
   - env: `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}`, `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}`
   - permissions: `contents: write`, `issues: write`, `pull-requests: write`

4. **.github/pull_request_template.md** — Checklist template:
   - Description of changes
   - Related issue number
   - Checklist: tests pass, lint passes, SPDX headers added, DCO sign-off (`git commit -s`), conventional commit format

5. **.github/CODEOWNERS** — `* @calmstudio/maintainers` (placeholder team).

All YAML workflow files need SPDX headers added as comments at the top (they're covered by dep5 but explicit headers are better practice when the format supports comments).

<task type="auto">
  <name>Task 2: Configure per-package semantic release and install release dependencies</name>
  <files>
    .releaserc.json,
    apps/studio/.releaserc.json,
    packages/calm-core/.releaserc.json,

  </files>
  <action>
1. **Install semantic release dev dependencies at workspace root:**
   ```bash
   pnpm add -D -w multi-semantic-release @semantic-release/changelog @semantic-release/git @semantic-release/npm @semantic-release/github
   ```

2. **Create `.releaserc.json` in each package** (apps/studio, packages/calm-core, packages/calmscript, packages/mcp-server, packages/extensions). Each file identical:
   ```json
   {
     "branches": ["main"],
     "plugins": [
       "@semantic-release/commit-analyzer",
       "@semantic-release/release-notes-generator",
       ["@semantic-release/changelog", { "changelogFile": "CHANGELOG.md" }],
       "@semantic-release/npm",
       ["@semantic-release/git", {
         "assets": ["package.json", "CHANGELOG.md"],
         "message": "chore(release): ${nextRelease.version} [skip ci]\n\n${nextRelease.notes}"
       }]
     ]
   }
   ```

   NOTE: Do NOT put `@semantic-release/github` in per-package configs (anti-pattern from research — creates a GitHub Release per package per push). Only use it if a root-level `.releaserc.json` is needed.

3. **Optionally create a root `.releaserc.json`** only if `multi-semantic-release` requires one. If not needed, skip it. The root workspace package has `private: true` and should not publish.

4. **Verify lockfile is updated** by running `pnpm install`.

All `.releaserc.json` files are covered by `.reuse/dep5` (JSON files cannot carry inline SPDX headers).
  </action>
