3. Export `async function layoutCalm(arch, pinnedIds, direction)`:
   - Build ELK graph from `arch.nodes` (filter out pinned IDs)
   - Set `elk.algorithm: 'layered'`, `elk.direction: direction`
   - Set spacing: `elk.layered.spacing.nodeNodeBetweenLayers: '100'`, `elk.spacing.nodeNode: '80'`
   - Use width: 160, height: 60 for all nodes (safe default matching Phase 2 node sizes)
   - Build edges from `arch.relationships`, filtering out edges where source OR destination is pinned
   - Call `await elk.layout(graph)`
   - Return Map of id -> {x, y} from layouted.children

Per RESEARCH Pitfall 7: Treat ELK graph as flat (no nested children) — sub-flow nesting handled by @xyflow/svelte parentId independently.

**In CalmCanvas.svelte:**
1. Add `onfileimport` callback prop: `let { onfileimport }: { onfileimport?: (content: string, filename: string) => void } = $props()` (alongside existing props)
2. Extend existing `handleDrop` function: after checking for palette node type (`application/calm-node-type`), add file drop handling — if `event.dataTransfer?.files[0]` exists and filename ends with `.json`, read file content via `file.text()` and call `onfileimport?.(content, file.name)`
3. Add pin toggle: on each node, add a small pin icon button visible on hover in the node wrapper. Store `pinned: boolean` in `node.data.pinned`. Toggle with `nodes = nodes.map(n => n.id === nodeId ? {...n, data: {...n.data, pinned: !n.data.pinned}} : n)`. Show a visual indicator (pushpin icon or dot) on pinned nodes.

**In +page.svelte:**
1. Import `layoutCalm` and `LayoutDirection` from `$lib/layout/elkLayout`
2. Import `tick` from `svelte`
3. Add module-level `let importError = $state<string | null>(null)` for error display (consumed by Plan 03 error banner)
4. Add `importCalmFile(content: string, filename?: string)` async function:
   - Try `JSON.parse(content)` as CalmArchitecture
   - Validate: must have `nodes` array — if not, set `importError = 'Invalid CALM JSON: missing nodes array'` and return early, canvas unchanged (per user decision: no partial load)
   - On JSON.parse error: set `importError = 'Malformed JSON: ' + error.message` and return early
   - Clear any previous error on success: `importError = null`
   - Call `pushSnapshot(nodes, edges)` before mutation
   - Call `applyFromJson(parsed)`
   - Call `const positionMap = await layoutCalm(parsed, new Set(), 'DOWN')` — no pinned nodes on import
   - Call `const projected = calmToFlow(parsed, positionMap)`
   - Set `nodes = projected.nodes; edges = projected.edges`
   - `await tick()` then call fitView (get it from useSvelteFlow or pass from CalmCanvas)
5. Add `runLayout(direction: LayoutDirection)` async function:
   - Collect pinned IDs from current nodes: `new Set(nodes.filter(n => n.data?.pinned).map(n => n.id))`
   - Get current model via `getModel()`
   - Call `layoutCalm(model, pinnedIds, direction)`
   - Build final positionMap: ELK results + pinned node current positions injected
   - Project via `calmToFlow(model, positionMap)`
   - Preserve pinned flag on projected nodes
   - `await tick()` then fitView
6. Add auto-layout button and direction dropdown to the canvas toolbar area (the existing `.toolbar` div at top-right):
   - A layout button (grid/arrange icon) that calls `runLayout('DOWN')` by default
   - A small dropdown next to it with three options: "Top to Bottom" (DOWN), "Left to Right" (RIGHT), "Hierarchical" (DOWN) — per user decision
   - Style consistently with existing `.toolbar-btn` class
7. Pass `onfileimport={importCalmFile}` to CalmCanvas component
8. Add Cmd+O keyboard shortcut to open file (temporarily using `<input type="file">` inline — proper File System Access API comes in Plan 02). On Cmd+O: create hidden input, accept `.json,.calm.json`, read content, call `importCalmFile`.
