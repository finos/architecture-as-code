  <action>
Add reverse sync logic in `+page.svelte`:

1. **Debounced reverse sync handler:**
   ```typescript
   let codeParseError = $state<string | null>(null);

   function handleCodeChange(newValue: string) {
     // This is called on every keystroke in CodeMirror.
     // Debounce: wait 400ms after last change before parsing.
     clearTimeout(codeChangeTimer);
     codeChangeTimer = setTimeout(() => {
       try {
         const parsed = JSON.parse(newValue) as CalmArchitecture;
         codeParseError = null;

         // Build position map from current nodes to preserve positions
         const positionMap = new Map<string, { x: number; y: number }>();
         for (const n of nodes) {
           if (n.data?.calmId) {
             positionMap.set(n.data.calmId as string, { ...n.position });
           }
         }

         // Push undo snapshot BEFORE applying
         pushSnapshot(nodes, edges);

         // Apply to canonical model (mutex prevents re-entry)
         const applied = applyFromJson(parsed);
         if (applied) {
           // Project back to Svelte Flow format, preserving positions
           const projected = calmToFlow(parsed, positionMap);
           nodes = projected.nodes;
           edges = projected.edges;
         }
       } catch (e) {
         codeParseError = (e as Error).message;
         // Canvas keeps last valid state — no update
       }
     }, 400);
   }
   let codeChangeTimer: ReturnType<typeof setTimeout>;
   ```

2. **Position preservation:** The positionMap captures current canvas positions keyed by calmId. When reverse sync projects CalmArchitecture back to Flow nodes, existing nodes keep their positions. New nodes (added via JSON) get default staggered positions via calmToFlow's built-in default.

3. **Unified undo/redo:** The existing handleUndo/handleRedo in CalmCanvas already restores snapshots. After restoring, they now also call `applyFromCanvas(nodes, edges)` (from Task 1). This means undo/redo updates the code panel automatically via forward sync.

   For code editor undo specifically: CodeMirror has its own internal undo (Cmd+Z when focused in editor). This is fine for keystroke-level undo within the editor. The app-level undo (Cmd+Z when canvas focused) undoes to the last debounce-committed state. This gives users two undo granularities which is expected behavior.

4. **Property mutations -> sync:** When the properties panel calls `updateNodeProperty` etc., the calmModel store updates the canonical model. The `$derived(getModelJson())` recalculates, updating CodePanel's value prop. The canvas needs to reflect property changes too.

   Add a `$effect` that watches the model and projects it to canvas nodes/edges when the model changes from a non-canvas source (properties panel):
   ```typescript
   // Watch for model changes from properties panel
   // The model store's $state triggers this derived
   const currentModel = $derived(getModel());

   // Track whether the last change came from canvas
   // If it did, don't re-project (the canvas already has the right state)
   let lastSyncSource = $state<'canvas' | 'code' | 'properties' | null>(null);
   ```

   Actually, simpler approach: Properties panel mutations update the model. Use `$effect` to detect model changes and re-derive nodes/edges ONLY when the sync source is properties:
   - After `applyFromCanvas`: set source='canvas' (no re-project needed)
   - After `applyFromJson` (code): set source='code' (already projected in handleCodeChange)
   - After property mutations: model changes via direct store update — need to re-project

   The cleanest approach per RESEARCH anti-pattern guidance: DO NOT use $effect for sync. Instead, make property mutation functions in the store also update a version counter. In +page.svelte, use $derived on the version to know when to re-project.

   Simplest correct approach: After any property panel mutation, the properties components should call a callback `onmutate` that the page provides, which re-projects:
   ```typescript
   function handlePropertyMutation() {
     const model = getModel();
     const positionMap = new Map<string, { x: number; y: number }>();
     for (const n of nodes) {
       if (n.data?.calmId) positionMap.set(n.data.calmId as string, { ...n.position });
     }
     const projected = calmToFlow(model, positionMap);
     nodes = projected.nodes;
     edges = projected.edges;
   }
   ```
   Pass this as `onmutate` prop through PropertiesPanel -> NodeProperties/EdgeProperties. Call it after each debounced store mutation.

5. **Cleanup:** Remove the `codeChangeTimer` on component destroy (Svelte 5 handles this if using $effect cleanup pattern, but since we're using plain setTimeout in a function, add cleanup in onDestroy or wrap in $effect if needed).
