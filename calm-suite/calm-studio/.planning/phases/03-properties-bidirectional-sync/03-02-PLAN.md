1. Create `InterfaceList.svelte` — inline list for CALM interfaces on a node.

Props:
```typescript
let { nodeId, interfaces = [] }: { nodeId: string; interfaces: CalmInterface[] } = $props();
```

Structure:
- Section header: "Interfaces" with count badge
- Each interface row: type dropdown (options: url, host-port, container-image, port, custom), value text input, delete button (X icon)
- On type change: call `updateInterface(nodeId, iface['unique-id'], { type: newType })`
- On value change: debounced (300ms) call `updateInterface(nodeId, iface['unique-id'], { value: newValue })`
- On delete click: call `removeInterface(nodeId, iface['unique-id'])`
- "+Add Interface" button at bottom: call `addInterface(nodeId, { 'unique-id': nanoid(), type: 'url', value: '' })`
- Import nanoid for generating interface unique-ids

Style: compact rows (32px height), small text (12px), matches existing monochrome palette. Dark mode via `:global(.dark)` selectors.

2. Create `CustomMetadata.svelte` — key-value pair editor.

Props:
```typescript
let { nodeId, metadata = {} }: { nodeId: string; metadata: Record<string, string> } = $props();
```

Structure:
- Section header: "Custom Properties" with count badge
- Each row: key text input (disabled after creation), value text input, delete button
- On value change: debounced call `addCustomMetadata(nodeId, key, newValue)` (addCustomMetadata is upsert)
- On delete: call `removeCustomMetadata(nodeId, key)`
- "+Add Property" button: adds empty row with focus on key input. On key blur, if key is non-empty, call `addCustomMetadata(nodeId, key, '')`
- Use local $state for the "new row" form; only commit to store when key is non-empty

Style: same compact row pattern as InterfaceList.

Add SPDX Apache 2.0 headers on both files.

Sections (vertical scroll):

- **Core fields**:
  - unique-id: read-only text display (gray, non-editable)
  - name: text input, on change call `updateNodeProperty(node.data.calmId, 'name', value)`. Also triggers pushSnapshot before the first mutation.
  - description: textarea (3 rows), on change call `updateNodeProperty(node.data.calmId, 'description', value)`
  - node-type: dropdown with all 9 CalmNodeType values plus "custom" option. On change call `updateNodeProperty(node.data.calmId, 'node-type', value)`. When "custom" selected, show a text input for the custom type string.
- **Interfaces section**: `<InterfaceList nodeId={node.data.calmId} interfaces={node.data.interfaces ?? []} />`
- **Controls section** (PROP-03 placeholder): disabled fieldset with "Controls" header and small "Coming in Phase 6" text. Gray background, not interactive.
- **Custom Properties section**: `<CustomMetadata nodeId={node.data.calmId} metadata={node.data.customMetadata ?? {}} />`

All text inputs have debounced handlers (300ms) — use local $state for input values, $effect with setTimeout/cleanup to commit to store.

CRITICAL: Before the FIRST property mutation in a user editing session, call `pushSnapshot()` from history store. Use a flag per-selection to avoid pushing snapshot on every keystroke — push once on first edit after selection changes.
