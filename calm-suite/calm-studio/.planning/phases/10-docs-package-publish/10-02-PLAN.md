2. Scaffold Docusaurus in the monorepo. Do NOT use `pnpm create docusaurus` (it creates unwanted nested structure). Instead, manually create the docs-site/ directory with a minimal Docusaurus setup:

   a. Create `docs-site/package.json`:
      - name: "calmstudio-docs"
      - private: true
      - scripts: start, build, clear, serve
      - dependencies: @docusaurus/core 3.9.2, @docusaurus/preset-classic 3.9.2, react, react-dom, prism-react-renderer, clsx
      - devDependencies: @docusaurus/module-type-aliases, @docusaurus/tsconfig, @docusaurus/types, typedoc, typedoc-plugin-markdown, docusaurus-plugin-typedoc, typescript

   b. Create `docs-site/docusaurus.config.ts`:
      - title: 'CalmStudio'
      - tagline: 'Visual CALM architecture editor — draw diagrams, get validated code'
      - favicon: 'img/favicon.ico'
      - url: 'https://finos.github.io'
      - baseUrl: '/calmstudio/'
      - organizationName: 'finos'
      - projectName: 'calmstudio'
      - trailingSlash: false
      - onBrokenLinks: 'throw'
      - onBrokenMarkdownLinks: 'warn'
      - plugins: docusaurus-plugin-typedoc with:
        - entryPoints: ['../packages/calm-core/src/index.ts']
        - tsconfig: '../packages/calm-core/tsconfig.json'
        - out: 'api'
        - sidebar: { categoryLabel: 'API Reference', position: 5 }
      - preset classic with:
        - docs: { sidebarPath: './sidebars.ts', editUrl: 'https://github.com/finos/calmstudio/tree/main/docs-site/' }
        - theme: { customCss: './src/css/custom.css' }
      - themeConfig:
        - navbar: title 'CalmStudio', logo (FINOS logo), items: Docs, API Reference, GitHub link
        - footer: links to FINOS CALM, AIGF, architecture-as-code, CalmStudio GitHub
        - prism: darkTheme, lightTheme
        - colorMode: respectPrefersColorScheme: true

   c. Create `docs-site/sidebars.ts`:
      ```typescript
      import type { SidebarsConfig } from '@docusaurus/plugin-content-docs';
      const sidebars: SidebarsConfig = {
        docs: [
          'intro',
          {
            type: 'category',
            label: 'Getting Started',
            items: ['getting-started/quick-start'],
          },
          {
            type: 'category',
            label: 'User Guide',
            items: ['user-guide/architecture-overview'],
          },
          {
            type: 'category',
            label: 'Developer Guide',
            items: [
              'developer-guide/extension-packs',
              'developer-guide/mcp-server',
              'developer-guide/contributing',
            ],
          },
          {
            type: 'category',
            label: 'ADRs',
            items: [{ type: 'autogenerated', dirName: 'adrs' }],
          },
        ],
      };
      export default sidebars;
      ```
      Note: API Reference sidebar is auto-generated by docusaurus-plugin-typedoc.

   d. Create `docs-site/src/css/custom.css` with FINOS-inspired colors:
      - Primary: #0033A0 (FINOS blue)
      - Dark primary: #0091EA (lighter blue for dark mode)
      - Clean, professional theme — no flashy gradients

   e. Create `docs-site/src/pages/index.tsx`:

      - Sub-hero: "Draw visually. Get validated architecture-as-code. Let AI generate via MCP."

      - CTA button: "Get Started" linking to /docs/getting-started/quick-start
      - Link to FINOS CALM ecosystem

   f. Create `docs-site/docs/intro.md`:

      - Key features list (canvas, bidirectional sync, extension packs, MCP, governance, C4 views)
      - Links to getting started, user guide, developer guide
      - Part of FINOS ecosystem with links

   g. Create placeholder content files so sidebar links resolve:

      - `docs-site/docs/developer-guide/contributing.md` — title + placeholder
      - `docs-site/docs/adrs/.gitkeep` — empty dir for ADRs (Plan 03 populates)

   h. Add a simple FINOS logo SVG to `docs-site/static/img/` — download from FINOS GitHub or use a text placeholder SVG that says "FINOS". Also add a basic `favicon.ico` placeholder.

1. Create `.github/workflows/docs.yml` following the pattern from RESEARCH.md:
   - Trigger: push to main branch
   - Permissions: contents:read, pages:write, id-token:write (OIDC required for deploy-pages)
   - Concurrency group: "pages" with cancel-in-progress: false
   - Job 1 (build-docs):
     - checkout with fetch-depth: 0
     - pnpm/action-setup@v4 with version 9
     - actions/setup-node@v4 with node-version 20, cache pnpm
     - pnpm install --frozen-lockfile
     - Build calm-core first: `pnpm --filter @calmstudio/calm-core run build` (TypeDoc needs types)
     - Build docs: `pnpm --filter calmstudio-docs run build`
     - Upload artifact: actions/upload-pages-artifact@v3 with path: docs-site/build
   - Job 2 (deploy-docs):
     - needs: build-docs
     - environment: github-pages with URL from deployment output
     - actions/deploy-pages@v4

2. Verify the workflow YAML is valid:
   - Check indentation, job names, step names
   - Ensure id-token: write is present (Pitfall 5 from research)
