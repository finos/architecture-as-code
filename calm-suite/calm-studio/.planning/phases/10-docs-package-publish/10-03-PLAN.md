ADRs to write:

1. **0001-use-svelte-5-over-react.md**: Context — needed modern, performant UI framework for visual editor. Options — React 18, Svelte 5, Vue 3. Chosen — Svelte 5 for lighter weight, better DX with runes, native reactivity. Good — smaller bundle, less boilerplate, Svelte Flow works well. Bad — smaller ecosystem than React.

2. **0002-use-svelte-flow-as-canvas.md**: Context — needed graph/diagram canvas library with typed nodes, edges, containment. Options — Excalidraw (freehand), Svelte Flow (typed graph), Custom canvas (Canvas 2D/WebGL). Chosen — Svelte Flow (port of React Flow) for typed graph editing with handles, containment, zoom/pan. Good — native node types, handles, built-in layout hooks. Bad — drop Excalidraw freehand sketching (was original prototype).

3. **0003-use-elk-for-auto-layout.md**: Context — need automatic layout for imported CALM JSON and re-layout operations. Options — ELK.js, dagre, d3-hierarchy. Chosen — ELK.js for hierarchical layout with LR/TB presets. Good — handles containment/nesting, multiple layout algorithms. Bad — larger bundle than dagre.

4. **0004-domain-oriented-control-keys.md**: Context — CALM spec controls need key naming convention. Options — Framework-prefixed keys (`aigf-data-protection`), Domain-oriented keys (`data-protection`). Chosen — Domain-oriented per CALM spec maintainer guidance. Framework IDs go in config-url only. Good — spec-aligned, reusable across frameworks. Neutral — requires AIR-ID mapping layer.

5. **0005-defer-calmscript-dsl.md**: Context — planned CalmScript DSL for human-readable architecture definitions. Options — Build DSL now, Defer DSL. Chosen — Defer. MCP server (21 tools) solves AI-assisted architecture generation more reliably. Good — avoid premature abstraction, MCP proven working. Bad — no human-friendly DSL yet. Status: accepted (revisit after real-world MCP usage).

6. **0006-extension-pack-system.md**: Context — 9 built-in CALM node types insufficient for cloud/K8s/AI architectures. Options — Hardcode all types, Plugin system, Extension packs. Chosen — Extension packs with PackDefinition type. Good — 7 packs shipped (60+ node types), easy to add more. Neutral — all packs bundled (no lazy loading yet).

7. **0007-mcp-server-as-primary-ai-integration.md**: Context — need AI tools to generate and modify CALM architectures. Options — REST API, CLI tool, MCP server. Chosen — MCP server for native Claude Code/VS Code Copilot integration. Good — 21 tools, works with Claude Code out of the box, stateless tool calls. Neutral — requires MCP-capable AI client.

8. **0008-docusaurus-for-documentation.md**: Context — need documentation site for guides, API reference, ADRs. Options — Docusaurus, Nextra, MkDocs, Starlight. Chosen — Docusaurus 3.x, FINOS ecosystem standard. Good — React+MDX, TypeDoc plugin, versioning, search. Neutral — heavier than Starlight but FINOS-aligned.

9. **0009-tsup-for-calm-core-packaging.md**: Context — calm-core needs publishing as standalone npm package with ESM+CJS dual output. Options — tsc only, tsup, rollup. Chosen — tsup for zero-config esbuild-powered dual output with .d.ts. Good — fast builds, correct output extensions, tree-shaking. Neutral — adds devDependency.

10. **0010-github-pages-hosting.md**: Context — need hosting for documentation site. Options — GitHub Pages, Vercel, Netlify. Chosen — GitHub Pages with Actions-based deployment. Good — free, FINOS standard, OIDC-verified deploy. Neutral — no server-side features (fine for static docs).
