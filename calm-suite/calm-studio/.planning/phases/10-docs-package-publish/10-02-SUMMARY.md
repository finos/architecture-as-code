## Decisions Made

- Used typedoc 0.28.x instead of 0.27.x from the research doc — required for TypeScript 5.9.x peer compatibility
- Set TypeDoc `out: 'docs/api'` (inside docs folder) so sidebar IDs resolve as `api/...` not `../api/...`
- `sidebars.ts` loads `typedoc-sidebar.cjs` with try/catch since it's auto-generated and not committed to git
- ADRs sidebar uses static items (not autogenerated) to avoid empty-dir build errors before Plan 03 adds ADR content
- Updated deprecated `onBrokenMarkdownLinks` to `markdown.hooks.onBrokenMarkdownLinks` for Docusaurus 3.9.2

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed TypeDoc plugin sidebar config for docusaurus-plugin-typedoc v1.4.x**
- **Found during:** Task 1 (Scaffold Docusaurus site)
- **Issue:** The plan specified `sidebar: { categoryLabel: 'API Reference', position: 5 }` but in v1.4.x the valid keys are `autoConfiguration, pretty, typescript, deprecatedItemClassName`
- **Fix:** Changed sidebar option to `{ autoConfiguration: true, pretty: true }`
- **Files modified:** docs-site/docusaurus.config.ts
- **Verification:** Build succeeded, no sidebar config error
- **Committed in:** 4379149

**2. [Rule 1 - Bug] Fixed empty ADRs directory causing Docusaurus build failure**
- **Found during:** Task 1 (Scaffold Docusaurus site)
- **Issue:** `{ type: 'autogenerated', dirName: 'adrs' }` in sidebars.ts fails with "No docs found in adrs: can't auto-generate a sidebar" on empty directory
- **Fix:** Added `docs/adrs/index.md` placeholder and changed sidebar to static items
- **Files modified:** docs-site/sidebars.ts, docs-site/docs/adrs/index.md (created)
- **Verification:** Build succeeded, no empty-dir error
- **Committed in:** 4379149

**3. [Rule 1 - Bug] Fixed deprecated onBrokenMarkdownLinks config key**
- **Found during:** Task 1 (Docusaurus build)
- **Issue:** Top-level `onBrokenMarkdownLinks: 'warn'` is deprecated in Docusaurus 3.9.2
- **Fix:** Moved to `markdown.hooks.onBrokenMarkdownLinks: 'warn'`
- **Files modified:** docs-site/docusaurus.config.ts
- **Verification:** No deprecation warning in build output
- **Committed in:** 4379149

**4. [Rule 1 - Bug] Fixed navbar API Reference link using non-existent sidebarId**
- **Found during:** Task 1 (Docusaurus SSG build)
- **Issue:** `sidebarId: 'typedoc'` navbar item fails with "Can't find any sidebar with id typedoc in version current"
- **Fix:** Changed to `to: '/docs/api/'` direct link to the TypeDoc-generated index page
- **Files modified:** docs-site/docusaurus.config.ts
- **Verification:** Build succeeded, all 35 pages render
- **Committed in:** 4379149

**5. [Rule 3 - Blocking] Updated typedoc version from ^0.27.0 to ^0.28.0**
- **Found during:** Task 1 (pnpm install)
- **Issue:** typedoc 0.27.x does not support TypeScript 5.9.3 (peer dependency constraint ends at 5.8.x); typedoc-plugin-markdown 4.10.0 requires typedoc 0.28.x
- **Fix:** Updated docs-site/package.json to use typedoc ^0.28.0, typedoc-plugin-markdown ^4.10.0, docusaurus-plugin-typedoc ^1.4.2
- **Files modified:** docs-site/package.json, pnpm-lock.yaml
- **Verification:** pnpm install completes with no peer dependency errors
- **Committed in:** 4379149
